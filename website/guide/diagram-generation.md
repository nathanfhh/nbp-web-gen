圖表模式專門用於生成技術圖表、流程圖、架構圖等視覺化內容。

## 基本用法

1. 選擇「圖表」模式
2. 描述你需要的圖表內容
3. 選擇圖表類型
4. 點擊「生成」

## 圖表類型範例

以下是幾個常用的圖表類型，實際上你可以嘗試生成各種不同類型的圖表：

### 流程圖

```
使用者登入流程：
1. 輸入帳號密碼
2. 驗證資料
3. 成功則進入首頁
4. 失敗則顯示錯誤訊息並重試
```

### 架構圖

```
三層式架構：
- 展示層：React 前端
- 邏輯層：Node.js API 伺服器
- 資料層：PostgreSQL 資料庫
```

### 時序圖

```
API 請求時序：
Client -> API Gateway -> Auth Service -> Database
```

::: tip 提示
除了以上範例，你也可以嘗試生成心智圖、組織架構圖、ER 圖、甘特圖等各種類型的圖表。只要在 Prompt 中清楚描述你需要的內容即可。
:::

## 圖表選項說明

### 畫質

設定輸出圖片的解析度：1K、2K、4K。

### 圖表類型

選擇要生成的圖表類型：

- **不指定**：讓 AI 自動判斷
- **流程圖**：展示流程步驟
- **架構圖**：系統架構示意
- **網路拓撲**：網路連接關係
- **資料庫**：資料庫結構（ER 圖等）
- **線框圖**：UI/UX 設計草圖
- **心智圖**：思維導圖
- **時序圖**：時間順序的互動流程

### 視覺風格

設定圖表的視覺呈現方式：

- **不指定**：讓 AI 自動決定
- **清爽**：乾淨的線條和色塊
- **手繪**：類似白板塗鴉的效果
- **技術**：專業技術文件風格

### 版面配置

設定圖表的整體排版方向：

- **不指定**：讓 AI 自動決定
- **水平**：從左到右排列
- **垂直**：從上到下排列
- **階層**：樹狀階層結構
- **環形**：圓形放射狀排列

### 複雜度

控制圖表的詳細程度：

- **不指定**：讓 AI 自動決定
- **簡單**：只顯示主要元素
- **詳細**：包含較多細節
- **全面**：完整詳盡的呈現

### 標註等級

設定圖表上的文字標註數量：

- **不指定**：讓 AI 自動決定
- **精簡標註**：只標註關鍵元素
- **詳細標註**：每個元素都有說明

## 提示詞技巧

### 明確結構

清楚描述圖表的層級和關係：

```
清楚標示：
- 主要節點用大方塊
- 次要節點用小方塊
- 使用箭頭表示資料流向
- 用不同顏色區分不同類型
```

### 加入說明

```
在圖表下方加入圖例說明
每個節點都要有中文標籤
```
