# 故事模式

故事模式讓你創作多步驟連續的視覺敘事，維持角色和風格的一致性。

## 範例：三隻狗的美西公路旅行

以下示範如何生成一組 4 步驟的故事，講述三隻狗的美西國家公園之旅：

### 步驟 1：選擇模式並設定步驟數

1. 選擇「故事」模式
2. 在步驟數量中選擇「4 步」

### 步驟 2：撰寫故事 Prompt

在 Prompt 中描述角色設定與故事流程：

```
角色：三隻好朋友——一隻橘色柴犬、一隻灰白色哈士奇、一隻黑白色邊境牧羊犬
風格：皮克斯 3D 動畫風格，溫暖明亮的色調

故事流程：
1. 三隻狗興奮地從飛機走出來，背景是洛杉磯國際機場 (LAX) 的標誌性建築
2. 他們站在乾燥的沙漠岩石前，背景是約書亞樹國家公園的獨特植被
3. 他們站在大峽谷的觀景台上，壯麗的峽谷風景在身後展開
4. 夕陽下，三隻狗坐在海邊沙灘上，望著太平洋的日落，結束這趟美好的旅程
```

::: tip 提示
將角色描述和故事流程寫在同一個 Prompt 中，系統會根據步驟數自動對應到每一步的生成。
:::

### 步驟 3：生成故事

點擊「生成」按鈕，系統會依序生成故事圖片。

![故事生成結果](/images/story-result.webp)

::: warning 關於生成數量
由於圖片模型會自行判斷每個步驟需要幾張圖片，實際生成的數量可能會超過你設定的步驟數。例如設定 4 步驟，可能會產出 8-10 張相似的圖片。這是目前模型的技術限制，建議在生成完成後自行挑選最符合需求的圖片。
:::

## 進階用法

### 使用參考圖片

如果你希望角色外觀更一致，可以上傳參考圖片：

1. 在參考圖片區域上傳角色圖片
2. 系統會在每一步都參考這張圖片
3. 同時也會參考前一步的生成結果，確保視覺連貫

### Prompt 結構建議

好的故事 Prompt 通常包含：

```
角色：（詳細描述外觀特徵）
風格：（畫風、色調）

故事流程：
1. （第一步場景描述）
2. （第二步場景描述）
3. （第三步場景描述）
4. （第四步場景描述）
```

### 使用角色庫

如果你已經在角色庫中建立了角色，可以直接選用：

1. 在「我的角色」區域選擇角色
2. 角色會自動加入參考圖片
3. AI 會根據角色設定生成一致的外觀

詳細說明請參閱 [角色庫](./character-library)。

## 生成流程

故事模式採用**逐步生成**，每一步都會參考：
- 你上傳的原始參考圖片（角色一致性）
- 前一步的生成結果（視覺連貫性）

這樣可以確保角色外觀穩定，同時場景轉換自然流暢。

## 故事選項說明

### 畫質

設定輸出圖片的解析度：1K、2K、4K。

### 分鏡步驟

可選擇 2-8 步。注意：實際生成張數可能因 AI 詮釋而有所不同。

### 序列類型

指定故事的敘事類型：

- **不指定**：讓 AI 自動判斷
- **故事**：傳統敘事結構
- **流程**：步驟式說明
- **教學**：教學導向內容
- **時間軸**：按時間順序展現變化

### 視覺一致性

控制各步驟間的視覺風格：

- **不指定**：讓 AI 自動決定
- **一致**：保持風格統一
- **演進**：允許風格漸變

### 轉場效果

設定步驟間的轉場風格：

- **不指定**：讓 AI 自動決定
- **順暢**：自然過渡
- **戲劇化**：強烈對比變化
- **淡出**：柔和的淡入淡出

### 輸出格式

指定圖片的呈現方式：

- **不指定**：讓 AI 自動決定
- **分鏡板**：類似漫畫分格的排版
- **單張**：每步一張獨立圖片

## 匯出選項

完成的故事可以：

- 下載為 ZIP（所有圖片）
- 生成 PDF
